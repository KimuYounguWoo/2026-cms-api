

# 2026-CMS-API

## 주요 기능

### 1. 사용자 인증 및 권한 관리 (Auth)
* **JWT 기반 Stateless 인증**: 서버 세션을 공유하지 않는 무상태성 인증 구조를 채택하여 시스템 확장성을 확보했습니다.
* **역할 기반 접근 제어 (RBAC)**: Spring Security를 활용해 일반 사용자(`USER`)와 관리자(`ADMIN`)의 권한을 엄격히 분리했습니다.
* `/api/admin/**`: 관리자 전용 엔드포인트 (콘텐츠 승인 및 사용자 관리)
* `/api/auth/**`: 비로그인 사용자 접근 허용 (인증 및 회원가입)


* **비밀번호 보안**: `BCryptPasswordEncoder`를 적용하여 단방향 해시 암호화를 통해 사용자 정보를 안전하게 보호합니다.

### 2. 콘텐츠 관리 시스템 로직

* **게시글 CRUD: 게시글의 생명주기(작성, 조회, 수정, 삭제) 관리 기능을 제공합니다.
* **데이터 캡슐화**: `Entity`와 `DTO`를 명확히 분리하여 내부 데이터 구조의 노출을 최소화하고 API의 하위 호환성을 유지했습니다.

### 3. 유연한 보안 및 인프라 설정

* **독립적 보안 필터 체인**: 서비스 API, H2 Console, Actuator 각각에 최적화된 독립적 보안 정책을 적용하여 설정 간 간섭을 방지했습니다.
* **시스템 모니터링 보호**: Actuator 엔드포인트를 관리자 권한으로 제한하여 시스템 내부 정보 유출을 차단했습니다.

### 4. Redis 활용 및 성능 최적화

* **분산 환경 토큰 관리**: Redis를 활용하여 Refresh Token 저장 및 관리를 수행, JWT의 보안 취약점을 보완했습니다.
* **Write-Behind 기반 조회수 시스템**:
* 조회 시마다 발생하는 DB Write 부하를 방지하기 위해 Redis에서 실시간 카운팅을 수행합니다.
* **Spring Scheduler**와 **JdbcTemplate.batchUpdate**를 결합하여 1시간 주기로 Redis의 데이터를 DB에 일괄 반영하는 효율적인 아키텍처를 구현했습니다.



### 5. 안정적인 예외 처리 및 유효성 검사

* **Global Exception Handling**: `@RestControllerAdvice`를 도입하여 모든 예외 상황을 중앙 집중식으로 관리하고 일관된 에러 응답 포맷(`ErrorResponse`)을 제공합니다.
* **정교한 검증 및 대응**: 토큰 만료, 변조, 미지원 형식 등을 세분화하여 처리하며, DB 제약 조건 위반(409)이나 경로 오류(404)에 대해서도 명확한 메시지를 반환합니다.
